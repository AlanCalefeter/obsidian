| Note : Nous utilisons le SQL (Structured Query Language), le langage standard pour communiquer avec les bases de données.  <br>  <br>L'ordre de création des tables (users, items, tags, item_tags, permissions) est crucial : si une relation existe entre deux tables, la table parente doit impérativement être créée avant la table enfant.  <br>  <br>Par exemple, la table shares possède une Clé Étrangère (Foreign Key -> FK) qui pointe vers la table users. On doit donc créer users en premier, sinon la base de données ne pourra pas établir le lien. |     |     |     |
